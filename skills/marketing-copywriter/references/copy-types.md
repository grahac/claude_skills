# Copy Types Reference

## Headlines & Taglines
```
# Formula: [Outcome] + [Timeframe/Ease] + [Without Pain Point]

Examples:
- "Launch your app in days, not months"
- "Write emails that actually get opened"
- "Finally, accounting software humans can use"

# Power words: Free, New, Now, Instantly, Proven, Easy, Simple, Fast
# Avoid: Revolutionary, Best-in-class, World-class (overused, vague)
```

## Value Propositions
```
# Formula: We help [audience] achieve [outcome] through [method]

Structure:
1. Headline: Clear benefit statement
2. Subhead: Who it's for + what it does
3. Bullets: 3-5 key benefits
4. Proof: Social proof or credibility

Example:
Headline: "Ship faster. Break less."
Subhead: "The deployment platform for teams who hate downtime"
Bullets:
- Zero-downtime deployments in one click
- Automatic rollbacks when things go wrong
- Real-time monitoring your whole team can understand
```

## Landing Pages
```
Structure (above the fold):
1. Headline: Primary benefit
2. Subheadline: Supporting context
3. Hero visual: Product in action
4. CTA: Clear, action-oriented button
5. Social proof: Logos, testimonials, numbers

Below the fold:
- Problem agitation
- Solution introduction
- Feature/benefit breakdown
- Testimonials with specifics
- FAQ addressing objections
- Final CTA with urgency
```

## Email Copy
```
Subject lines:
- Keep under 50 characters
- Create curiosity or promise value
- Personalize when genuine
- Test questions vs statements

Body structure:
1. Hook: First line earns the second
2. Problem: Acknowledge their situation
3. Solution: Introduce your answer
4. Proof: Why it works
5. CTA: One clear action

# Kill phrases: "I hope this finds you well", "Just checking in", "Per my last email"
```

## Ad Copy
```
Facebook/Instagram:
- Hook in first 3 words
- Use "you" language
- Keep primary text under 125 chars
- Clear CTA in copy AND button

Google Ads:
- Include keyword in headline
- Use numbers and specifics
- Highlight differentiators
- Strong CTA

General:
- Match ad to landing page message
- Test multiple variations
- Focus on one benefit per ad
```

## Product Descriptions
```
Structure:
1. Benefit-focused headline
2. Who it's for + primary use case
3. Key features as benefits
4. Specifications (when relevant)
5. Social proof snippet

Example:
"The notebook that thinks like you do"
Built for developers who hate context switching. Capture code, notes, and diagrams in one place.

- Write in markdown, see it formatted instantly
- Link notes together like your brain does
- Find anything in milliseconds, not minutes
- Works offline, syncs when you're ready
```

## CTAs (Calls to Action)
```
Strong CTAs:
- Start free trial (no "submit")
- Get instant access
- See it in action
- Claim your spot
- Download the guide

Make it specific:
- Not: "Learn more"
- Better: "See the full feature list"
- Best: "Watch the 2-minute demo"
```
